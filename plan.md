## Принятые решения

| Вопрос                      | Решение                                                                                                                                                                                                                         |
|-----------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Пространственная модель     | **Коридорная сетка.** Игрок ходит по коридорам. Комнаты — за дверями в стенах коридоров. Сетка NxN: каждая ячейка 6×6 м (комната 4×4 м в центре + коридор 2 м по периметру). DFS-maze определяет, какие коридоры открыты.       |
| Двери vs. проходы           | **Разные вещи.** Проход = убранная стена между ячейками (maze). Дверь = проём в стене коридора, ведущий в комнату (raymarching). Дверь ставится в стене коридора со стороны одного из открытых проходов ячейки (выбор по hash). |
| Вход в комнату              | **Fullscreen toggle.** При входе — fullscreen quad. Не stencil-порталы.                                                                                                                                                         |
| Координаты камеры в шейдере | **shaderPos = worldPos − roomCenter.** Комната в шейдере: (0,0,0) = центр, диапазон ±2 м по XZ, 0…2.5 м по Y.                                                                                                                   |
| Подглядеть снаружи          | **Превью-текстура 64×64.** Камера превью: позиция у двери (0.5 м внутрь комнаты), смотрит на центр.                                                                                                                             |
| Коллизия в коридорах        | **Split-axis.** Двигаем по X — проверяем стены — откатываем если стена. Затем по Z. Даёт скольжение вдоль стен бесплатно.                                                                                                       |
| Коллизия в комнате          | **Bounds only.** Box 4×4×2.5 м. Не по SDF.                                                                                                                                                                                      |
| Большие этажи               | **Chunked corridors.** Меши только в радиусе ~20 ячеек от игрока.                                                                                                                                                               |
| Производительность шейдеров | **Adaptive resolution.** Raymarching в буфер 0.5–1.0× экрана, upscale. В шаблоне с этапа 3.                                                                                                                                     |
| Этажи 6–9 (1 комната)       | **Так задумано.** Один коридор, одна дверь. Плавный вход.                                                                                                                                                                       |
| Этаж 0                      | **Вестибюль.** 6×6×2.5 м. Пустая комната, лестница вверх у дальней стены. Первое, что видит игрок.                                                                                                                              |
| Этажи 1–5                   | **Прямой коридор 30×4×2.5 м.** Голые стены, тусклый свет. Лестницы на обоих концах. Один тип, без вариаций.                                                                                                                     |
| Лестница (взаимодействие)   | **Триггер-зона.** Меш: светящаяся площадка на полу (emissive plane) + стрелка вверх/вниз. Игрок встаёт на площадку → fade → переход. Без геометрии ступенек.                                                                    |
| Переход между этажами       | **Fade.** Чёрный fade ~0.3 с → dispose старого этажа → генерация → fade-in.                                                                                                                                                     |
| Алгоритм лабиринта          | **DFS-maze** на сетке. Seed → spanning tree → убираем стены. Гарантированная связность.                                                                                                                                         |
| Стартовый экран             | **Этап 2a.** Простая форма с seed + «Войти».                                                                                                                                                                                    |
| Состояние ячеек             | **Map per floor.** `Map<"x,z", "valid" \| "empty">` — заполняется при генерации, используется для дверей и превью.                                                                                                              |
| Scene cleanup               | **При переходе** — dispose всего предыдущего этажа. В памяти один этаж.                                                                                                                                                         |
| Нормализация SDF            | **Этап 4e.** Центрирование + масштабирование объектов, чтобы влезали в комнату и выглядели не как каша.                                                                                                                         |
| Потолок этажей              | Бесконечно (chunking). Грамматика не усложняется после 500+.                                                                                                                                                                    |
| Звук                        | Вне скоупа. После этапа 10.                                                                                                                                                                                                     |

## Этап 1: Ходьба

**Цель:** Запустить, ходить от первого лица.

- Vite + Three.js, canvas fullscreen
- Камера от первого лица, WASD + мышь (pointer lock при клике)
- Escape — unlock
- Пустая комната (box 6×6×2.5 м) — пол, стены, потолок
- Физика: Y = 0, bounds check (split-axis: X, затем Z)

**MVP:** Запускаю → кликаю → хожу по комнате.

---

## Этап 2: Seed и коридоры

**Цель:** Процедурный лабиринт, seed задаёт мир.

### 2a: Seed-система и стартовый экран

- Seed: строка → числовой hash (mulberry32)
- Seedable PRNG
- Стартовый экран: поле seed + «Войти». Приоритет: URL `?seed=...` > input > дефолт
- Структура данных: Grid (side, cells, walls)

**MVP:** Ввожу seed → попадаю в мир.

### 2b: Коридорные меши и коллизия

- DFS-maze на сетке 3×3 (тест) → spanning tree, убираем стены
- Ячейка 6×6 м: комната 4×4 в центре (пока без шейдера — стена), коридор 2 м по периметру
- Меши: пол, потолок, стены коридоров
- Коллизия: split-axis (двигаем по X → проверка стен → откат; затем по Z). Скольжение вдоль стен

**MVP:** Хожу по лабиринту коридоров, не прохожу сквозь стены. Другой seed — другой лабиринт.

---

## Этап 3: Raymarching-комната

**Цель:** Зайти в комнату — увидеть шейдер.

### 3a: Raymarching-шаблон

- Fullscreen quad + ShaderMaterial
- Шаблон: uniform (uCameraPos, uCameraDir, uResolution, uTime), raymarch loop, слот `sceneSDF`
- Тестовая сцена: сфера
- **Adaptive resolution:** рендер в WebGLRenderTarget (scale 0.5–1.0×), upscale на экран

**MVP:** Полноэкранный raymarching со сферой, разрешение настраивается.

### 3b: Интеграция коридор ↔ комната

- Дверь (проём) в стене коридора одной из ячеек
- Детекция входа: позиция игрока в зоне проёма → `isInRoom = true`
- `isInRoom`: скрыть коридор, показать fullscreen quad
- Координатный мост: `shaderCameraPos = playerWorldPos − roomWorldCenter`
- Выход: `isInRoom = false`, обратно в коридор
- Коллизия внутри: bounds box 4×4×2.5 м

**MVP:** Иду по коридору → захожу через проём → вижу сферу → выхожу обратно.

---

## Этап 4: Грамматика — процедурные комнаты

**Цель:** Каждая комната уникальна, содержимое генерируется из seed.

### 4a: Один примитив

- Room_ID = hash(floor, x, z, seed)
- Генератор: один примитив (sphere / box / cylinder) по Room_ID
- Параметры (радиус, размер, позиция) из PRNG
- Вставка в шаблон (слот sceneSDF)
- Fallback при ошибке компиляции — gradient

**MVP:** Разные комнаты — разные фигуры.

### 4b: Комбинации

- Дерево глубины 2: `combine(A, B)` — min, max, smoothMin
- Генератор дерева → строка GLSL

**MVP:** Комнаты с двумя объектами (union / intersection / blend).

### 4c: Модификаторы

- twist, bend, repeat
- Параметры из PRNG

**MVP:** Искажённые формы в комнатах.

### 4d: Валидация и retry

- Offscreen рендер 16×16, средняя яркость < 0.02 → «чёрный экран»
- Retry: Room_ID + 1, до 20 попыток (см. ADR-0004)
- При провале — ячейка «пустая» (глухая стена, нет двери)
- Результат → `Map<"x,z", "valid" | "empty">` на этаж

**MVP:** Нет чёрных комнат — либо рабочий шейдер, либо стена.

### 4e: Нормализация SDF

- Центрирование: bounding-оценка дерева → смещение к (0, 0, 0)
- Масштабирование: объект вписывается в сферу радиуса ~1.5 м (не вылезает за стены комнаты)
- Ограничение пропорций примитивов (нет плоских дисков или игл)

**MVP:** Объекты в комнатах визуально осмысленные, расположены по центру, вмещаются в пространство.

---

## Этап 5: Этажи и лестницы

**Цель:** Вертикальное путешествие.

### 5a: Многоэтажность (6+)

- Структура: World → текущий Floor
- Формула (этажи 6+): `rooms = floor(1 + 0.01 * floor²)`, side = ceil(√rooms)
- Генерация этажа: сетка по формуле + DFS-maze + коридорные меши
- Dispose старого этажа при переходе (мгновенный, без fade — для отладки)
- Данные этажа: grid, maze, `Map<"x,z", status>`

**MVP:** Могу переключаться между этажами 6, 7, 8… (debug-кнопка). Каждый — свой лабиринт и комнаты.

### 5b: Лестницы

- Меш: emissive plane на полу + стрелка (вверх / вниз)
- Позиция: по hash(floor), угол или край сетки. Одна лестница на этаж
- Триггер-зона: игрок встаёт на площадку → fade-out 0.3 с → dispose → генерация нового этажа → fade-in
- При подъёме: спавн у лестницы нового этажа. При спуске — аналогично

**MVP:** Иду по коридору → нахожу лестницу → встаю → поднимаюсь. Обратно — спускаюсь.

### 5c: Этажи 0 и 1–5

- Этаж 0: вестибюль 6×6×2.5 м, одна лестница вверх у дальней стены
- Этажи 1–5: прямой коридор 30×4×2.5 м, лестницы на обоих концах
- UI: «Этаж N» в углу экрана

**MVP:** Старт на 0 → подъём через 1–5 → выход на этаж 6 с комнатами.

### 5d: Chunked corridors

- На этажах с сеткой > 20×20: меши коридоров только в радиусе R=20 **ячеек** от игрока (не чанков; см. TECH_SPEC §4)
- При движении — добавляем новые, убираем дальние (dispose)
- На малых этажах — всё

**MVP:** Этаж 100 (11×11) работает без просадок.

---

## Этап 6: Превью-текстуры

**Цель:** Видеть содержимое комнаты до входа.

- Offscreen рендер: WebGLRenderTarget 64×64
- Камера превью: 0.5 м внутрь от двери, смотрит на центр комнаты
- Результат → текстура на quad в проёме двери
- Генерация lazy: при приближении игрока к проёму
- Пустые ячейки: нет двери, нет превью

**MVP:** Иду по коридору → в дверях видны миниатюры шейдеров.

---

## Этап 7: Полная грамматика по зонам

**Цель:** Сложность растёт с этажом.

### 7a: Этажи 21–50

- Примитивы: + plane, torus
- Шум: noise3D
- Анимация: rotate_y(expr, uTime)
- Глубина дерева 2–3

**MVP:** Этаж 25 — шум и вращение.

### 7b: Этажи 51–150

- fbm, displace, fractal_repeat
- pulse, slide
- Глубина 3–4

**MVP:** Этаж 80 — фракталы и анимации.

### 7c: Этажи 151+

- menger, sierpinski, morph
- Цвет: градиенты HSV, пульсация hue
- Глубина 4–8

**MVP:** Этаж 200 — цветные математические галлюцинации.

---

## Этап 8: Кэш и сохранение

**Цель:** Не пересчитывать, помнить прогресс.

### 8a: IndexedDB кэш

- Store: `v{ver}:{seed}:{floor}:{x}:{z}` → GLSL-код или «пусто»
- При генерации — сначала кэш
- LRU при > 500 МБ (считаем размер записей при записи, храним суммарный размер в отдельном ключе)

**MVP:** Повторный заход в комнату — мгновенно.

### 8b: Сохранение позиции

- localStorage: seed, этаж, x, z
- Восстановление при перезагрузке
- Стартовый экран: приоритет URL > localStorage > дефолт

**MVP:** Перезагружаю — на том же этаже.

---

## Этап 9: Атмосфера

**Цель:** Лиминальный свет.

### 9a: Освещение коридоров

- Точечные лампы (PointLight), позиции по hash(floor, segment)
- Тёплый цвет, слабая интенсивность
- Этажи 1–5: мерцание (sin + noise по time)

### 9b: Освещение в шейдерах

- Ключевой свет от «двери» в шаблоне (uniform uDoorDir)
- Ambient минимальный

**MVP:** Тёплый полумрак в коридорах, направленный свет в комнатах.

---

## Этап 10: Полировка

**Цель:** Финальный продукт.

- Кнопка «Заархивировать» в комнате → IndexedDB (шейдер + метаданные)
- Меню по Escape: seed, лимит кэша
- WebGL fallback: сообщение при отсутствии
- Версионирование кэша (константа в коде, при смене грамматики — инкремент)
- Предгенерация: при входе на этаж — комнаты в радиусе 15 ячеек от точки входа; при приближении к лестнице — prewarm зоны следующего этажа

**MVP:** Полный цикл: seed → исследование → архив.
