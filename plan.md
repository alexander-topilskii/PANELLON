## Принятые решения


| Вопрос                      | Решение                                                                                                                                                                                                                         |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Пространственная модель     | **Коридорная сетка.** Игрок ходит по коридорам. Комнаты — за дверями в стенах коридоров. Сетка NxN: каждая ячейка 6×6 м (комната 4×4 м в центре + коридор 2 м по периметру). DFS-maze определяет, какие коридоры открыты.       |
| Двери vs. проходы           | **Разные вещи.** Проход = убранная стена между ячейками (maze). Дверь = проём в стене коридора, ведущий в комнату (raymarching). Дверь ставится в стене коридора со стороны одного из открытых проходов ячейки (выбор по hash). |
| Вход в комнату              | **Fullscreen toggle.** При входе — fullscreen quad. Не stencil-порталы.                                                                                                                                                         |
| Координаты камеры в шейдере | **shaderPos = worldPos − roomCenter.** Комната в шейдере: (0,0,0) = центр, диапазон ±2 м по XZ, 0…2.5 м по Y.                                                                                                                   |
| Подглядеть снаружи          | **Превью-текстура 64×64.** Камера превью: позиция у двери (0.5 м внутрь комнаты), смотрит на центр.                                                                                                                             |
| Коллизия в коридорах        | **Split-axis.** Двигаем по X — проверяем стены — откатываем если стена. Затем по Z. Даёт скольжение вдоль стен бесплатно.                                                                                                       |
| Коллизия в комнате          | **Bounds only.** Box 4×4×2.5 м. Не по SDF.                                                                                                                                                                                      |
| Большие этажи               | **Chunked corridors.** Меши только в радиусе ~20 ячеек от игрока.                                                                                                                                                               |
| Производительность шейдеров | **Adaptive resolution.** Raymarching в буфер 0.5–1.0× экрана, upscale. В шаблоне с фазы 4 (4.1).                                                                                                                                |
| Этажи 6–9 (1 комната)       | **Так задумано.** Один коридор, одна дверь. Плавный вход в мир raymarching-комнат.                                                                                                                                              |
| Этаж 0                      | **Главное лобби.** 20×20×2.5 м. Лестница вверх + телепорт-пады на 10/20/30/50/100/150/200/300/500.                                                                                                                              |
| Этажи 1–5                   | **Прямой коридор 30×4×2.5 м.** Голые стены, тусклый свет. Лестницы на обоих концах. Один тип, без вариаций.                                                                                                                     |
| Лестница (взаимодействие)   | **Триггер-зона.** Меш: светящаяся площадка на полу (emissive plane) + стрелка вверх/вниз. Игрок встаёт на площадку → fade → переход. Без геометрии ступенек.                                                                    |
| Лестница на этажах 6+       | **Две площадки (вверх/вниз) в разных ячейках.** Позиции по hash(floor). Обе на краю сетки, ячейки помечены `reservedCells` — в них нет дверей комнат.                                                                            |
| Переход между этажами       | **Fade.** Чёрный fade ~0.3 с → dispose старого этажа → генерация → fade-in.                                                                                                                                                     |
| Алгоритм лабиринта          | **Eager DFS-maze + chunked mesh.** Топология лабиринта генерируется целиком (компактные данные). Меши строятся только для активного радиуса. См. ADR-0007.                                                                      |
| Room_ID                     | **hash(version, globalSeed, floor, x, z).** `version` включён для изоляции несовместимых грамматик. См. DETERMINISM_CONTRACT.                                                                                                   |
| Стартовый экран             | **Фаза 1 (1.2).** Простая форма с seed + «Войти».                                                                                                                                                                               |
| Состояние ячеек             | **Map per floor.** `Map<"x,z", "valid" | "empty">` — заполняется при генерации, используется для дверей и превью. In-memory до фазы 6, затем IndexedDB.                                                                         |
| Scene cleanup               | **При переходе** — dispose всего предыдущего этажа. В памяти один этаж.                                                                                                                                                         |
| Нормализация SDF            | **Фаза 4 (4.7).** Центрирование + масштабирование объектов, чтобы влезали в комнату и выглядели не как каша.                                                                                                                    |
| Потолок этажей              | **9999 (v1).** Грамматика не усложняется после 500+ — комбинаторное разнообразие на глубине 5–8 достаточно.                                                                                                                     |
| WebGL                       | **Требуется WebGL2.** При отсутствии — экран с сообщением.                                                                                                                                                                      |
| Миникарта                   | **Canvas 2D overlay.** Круглая мини-карта 160px (bottom-right), toggle на полный экран (500px, по центру) по M. Показывает стены лабиринта, двери, лестницы, телепорты, позицию и направление игрока.                             |
| Телепорт-пады               | **Синие светящиеся платформы** со Sprite-метками номера этажа. Этаж 0: 10/20/30/50/100/150/200/300/500. Каждый 100-й — хаб с телепортами на ±10 этажей.                                                                          |
| Хаб-этажи (100, 200...)     | **Лобби 20×20 м.** Тип `lobby`. Лестницы вверх/вниз + телепорты на ближайшие десятки (±90 от базы). Позволяют быстрый доступ в окрестности.                                                                                     |
| Reserved cells              | **Ячейки под лестницы/телепорты** не получают дверей комнат. `FloorDescriptor.reservedCells` → `buildRoomWalls` пропускает генерацию двери для этих ячеек.                                                                       |
| Шейдер-дебаг                | **По клавише I** — fullscreen overlay с исходным GLSL кодом текущей комнаты. Помогает в отладке генерации.                                                                                                                       |
| Door trigger tuning         | **Trigger depth уменьшен с 0.6 до 0.25 м**, playerBox с 0.4 до 0.3 м. Предотвращает ложный вход в комнату при проходе мимо двери по коридору.                                                                                  |
| Merge geometries            | **`mergeGeometries` из three/addons.** Все wall-сегменты коридоров и room-walls объединяются в 1 mesh на этаж. Снижает draw calls с тысяч до единиц.                                                                            |
| Spatial hash (collision)    | **Uniform grid (`SpatialHash`, CELL_SIZE bucket).** `resolveWallCollisions` запрашивает 9 соседних бакетов — O(1) вместо O(N). Файл: `src/input/spatial-hash.ts`.                                                               |
| Light culling               | **Distance-based visibility.** Каждый кадр: `light.visible = dist² < LIGHT_RADIUS²`. Радиус 30 м (~5 ячеек). На этаже 150+ отключает ~90 % ламп, снижая forward-pass нагрузку.                                                 |
| Minimap cache               | **Offscreen canvas.** Стены лабиринта рисуются один раз при `setFloorData()` в offscreen canvas. Каждый кадр — `drawImage` вместо итерации по N² ячейкам.                                                                      |
| Spatial hash (triggers)     | **Дверные trigger-зоны** индексируются в `SpatialHash`. `updateCorridor` проверяет только двери из 9 соседних бакетов, а не все на этаже.                                                                                       |
| Звук                        | Вне скоупа. После фазы 7.                                                                                                                                                                                                       |
| Превью-текстуры             | **64×64 WebGLRenderTarget на дверь.** Отдельный RT на каждый превью (не пул). Камера 1.5 м внутрь комнаты, направлена на центр. Lazy-генерация при ≤ 2 ячейки, max 2 compile/frame.                                               |
| noise3D в шейдерах          | **Value noise** через permutation-хеш (mod-арифметика). Без texture lookup. Используется в displace и fbm.                                                                                                                       |
| Цвет комнат (HSV)           | **Tier-зависимый.** Tier 1 = бежевый монохром, Tier 2 = лёгкий оттенок по normal, Tier 3 = hue по позиции + время, Tier 4 = полный HSV + sin-пульсация. Через `uColorSeed` и `uTier` uniforms.                                  |
| Menger/Sierpinski            | **Stand-alone primitives**, не modifiers. Iterative SDF, до 6/8 итераций. Появляются на этажах 151+.                                                                                                                             |
| Вероятности грамматики       | **Взвешенные категории:** combiner 45%, modifier 15%, noise 12%, animation 12%, fractal 10%. Leaf-wrap: anim 25%, noise 15%.                                                                                                     |

## Баг-фиксы и извлечённые уроки

### BUG-001: Невозможно выйти из raymarching-комнаты

**Симптом:** игрок входил в комнату, но не мог вернуться в коридор.

**Причина:** `updateRoom()` сначала вызывал `applyRoomCollision()` (clamp позиции), а затем `checkRoomExit()`. Collision clamp ограничивал позицию до `softLimit = HALF_ROOM + 0.5`, а exit threshold был `HALF_ROOM + 0.3` — казалось бы, достаточно. Но мягкий лимит действовал только в узкой полосе двери (`±0.75 м`); при малейшем боковом отклонении включался жёсткий clamp `±(HALF_ROOM − ROOM_MARGIN) = ±1.75 м`, не пуская к порогу 2.3 м.

**Урок: проверка перехода между пространствами должна выполняться до ограничения позиции.** Иначе clamp «съедает» движение, и условие выхода никогда не выполняется.

**Фикс:** `checkRoomExit()` теперь вызывается на «сырой» позиции сразу после применения движения, до `applyRoomCollision()`.

### BUG-002: Миникарта — игрок «проходит сквозь стены»

**Симптом:** при входе в комнату точка игрока на миникарте прыгала к центру карты и двигалась в room-local координатах, не совпадая с реальной позицией в мире.

**Причина:** `enterRoom()` переводит `player.position` в room-local координаты (`worldPos − roomCenter`), а миникарта читала `player.position` напрямую, считая его мировой позицией.

**Урок: при наличии двух систем координат (мировая vs. room-local) все внешние потребители позиции должны получать её через единый геттер**, который знает текущий контекст.

**Фикс:** добавлен `FloorManager.getWorldPosition()` — возвращает `localPos + roomCenter` при `inRoom`, иначе — `player.position`. Миникарта использует его вместо прямого чтения.

### BUG-003: Лестницы и телепорты попадали внутрь шейдер-комнат

**Симптом:** на процедурных этажах ячейка с лестницей могла иметь дверь в raymarching-комнату, и игрок «засасывался» в комнату вместо использования лестницы.

**Причина:** `buildRoomWalls` генерировал двери для всех ячеек без исключений, включая ту, где стояла лестница.

**Урок: ячейки с инфраструктурными объектами (лестницы, телепорты) должны быть явно исключены из генерации контента.** Механизм `reservedCells` в дескрипторе этажа передаёт список «занятых» ячеек в builder.

**Фикс:** `FloorDescriptor.reservedCells` → `buildRoomWalls` получает `Set` зарезервированных ячеек, в них `doorWall = 0` (без двери).

### BUG-004: Ложный вход в комнату при проходе мимо двери

**Симптом:** просто проходя по коридору мимо двери, игрок мог случайно попасть в комнату.

**Причина:** trigger zone двери выступала на 0.6 м в коридор (шириной 2 м), а playerBox имел размер 0.4 м. Суммарно — зона перехвата 1.0 м на коридор шириной 2 м — слишком много.

**Урок: trigger-зоны должны быть соразмерны с геометрией окружения.** В коридоре шириной 2 м глубина trigger > 0.3 м уже критична.

**Фикс:** trigger depth уменьшен до 0.25 м, playerBox — до 0.3 м. Игрок входит в комнату только при целенаправленном движении к двери.

## Оптимизация производительности (этажи 150+)

**Симптом:** после ~150 этажа FPS падал до <20. Размер сетки лабиринта растёт с номером этажа (формула `gridSide`), на 150-м — 25×25 = 625 ячеек, тысячи mesh/wall/light объектов.

**Анализ узких мест:**

| Бутылочное горлышко            | Было                                      | Стало                                  | Эффект                              |
| ------------------------------ | ----------------------------------------- | -------------------------------------- | ----------------------------------- |
| Draw calls (wall meshes)       | Каждый wall-сегмент — отдельный Mesh      | `mergeGeometries` → 1–3 mesh на этаж   | Draw calls: ~2000 → ~5              |
| Collision (`resolveWallCollisions`) | Линейный перебор всех `wallBoxes` (O(N)) | `SpatialHash` → query 9 бакетов (O(1)) | Frame time: ~4 ms → ~0.1 ms        |
| Lights (PointLight)            | Все лампы активны одновременно            | `light.visible` по дистанции (30 м)    | Активных ламп: ~150 → ~15          |
| Minimap redraw                 | Итерация по N² ячейкам каждый кадр        | Offscreen canvas + `drawImage`         | Minimap cost: ~2 ms → ~0.1 ms      |
| Door trigger scan              | Линейный перебор всех дверей (O(N))       | `SpatialHash` + Set → O(1) проверка    | Trigger check: ~0.5 ms → ~0.05 ms  |

**Файлы:**
- `src/input/spatial-hash.ts` — SpatialHash utility (uniform grid)
- `src/render/corridor-builder.ts` — mergeGeometries для wall-сегментов
- `src/render/door-builder.ts` — mergeGeometries для room walls
- `src/input/maze-collision.ts` — принимает SpatialHash вместо Box3[]
- `src/core/floor-manager.ts` — wallHash, doorHash, updateLights(), hasWallCollision
- `src/ui/minimap.ts` — offscreen maze cache

**Урок: на больших процедурных этажах узкие места — не алгоритм генерации, а per-frame стоимость.** Spatial indexing, geometry batching и distance culling — три столпа, решающие проблему в совокупности.
