# План реализации PANELLON

Поэтапный план. Каждый этап даёт **рабочий MVP** — можно запустить и что-то сделать.

---

## Анализ: что не хватает, что разбить

**Не хватает в плане:**
- Raymarching-шаблон (универсальный) — создание до грамматики
- Гибрид предпроверка + на лету (радиус N при входе)
- LRU для кэша при переполнении
- Пирамида: размер этажа по формуле (этаж 0 узкий, 6+ растёт)

**Сложные этапы — разбить:**
- Этап 4 (грамматика) — на 4a–4d
- Этап 7 (полная грамматика) — на 7a–7c

**Риски:**
- Этап 3: «безшовный вход» — проще всего: при `isInRoom` рендерить fullscreen quad с Raymarching вместо сцены с мешами; камера та же
- Этап 6: этажи 1–5 — разные типы требуют разной геометрии; начать с одного типа (коридор)

---

## Этап 1: Ходьба в пустой комнате

**Цель:** Запустить приложение, ходить от первого лица.

**Задачи:**
- Инициализация проекта (Vite + Three.js)
- Canvas на весь экран
- Камера от первого лица
- WASD + мышь (pointer lock при клике)
- Escape — выход из pointer lock
- Одна пустая комната (box 4×4×2.5 м) — стены, пол, потолок
- Простая физика: гравитация, коллизия с полом (без провала)

**MVP:** Запускаю → кликаю → хожу по комнате.

---

## Этап 2: Процедурный коридор

**Цель:** Коридор генерируется по seed.

**Задачи:**
- Seed-система: строка → числовой hash (mulberry32 или аналог)
- Один этаж: сетка 3×3 ячеек (упрощённо)
- Коридоры — процедурные меши (стены между ячейками)
- **Коллизия со стенами** (иначе пройдёшь сквозь)
- Детерминизм: один seed = один коридор
- Seed из URL `?seed=...` или дефолт

**MVP:** Меняю seed в URL → вижу другой коридор. Не прохожу сквозь стены.

---

## Этап 3: Первая комната с Raymarching

**Цель:** Одна комната с шейдером, можно зайти.

**Задачи:**
- **Raymarching-шаблон** (универсальный): uniform camera, raymarch loop, слот для `sceneSDF`. Создать один раз.
- Один проём в стене коридора (дверная коробка)
- За проёмом — объём 4×4×2.5 м
- Raymarching: **жёстко зашитая** сфера в шаблоне
- Безшовный вход: камера в том же мире; при позиции «в комнате» — рендер Raymarching вместо мешей (или stencil mask). Проще: при входе переключаем рендер на fullscreen quad с Raymarching.
- Коллизия: не выходить за пределы комнаты, не проваливаться

**MVP:** Иду по коридору → вижу проём → заходим → вижу сферу в шейдере.

---

## Этап 4: Грамматика — процедурные комнаты

**Цель:** Содержимое комнаты генерируется по Room_ID.

> Разбито на подэтапы 4a–4d — грамматика объёмная.

### 4a: Шаблон + один примитив

- Room_ID = hash(floor, grid_x, grid_z, seed)
- Генератор: **один** примитив (sphere ИЛИ box ИЛИ cylinder) — выбор по Room_ID % 3
- Параметры (радиус, размер) из seedable PRNG
- Вставка в Raymarching-шаблон (слот `sceneSDF`)
- Fallback при ошибке компиляции — простой gradient

**MVP:** Разные примитивы в разных ячейках.

### 4b: Комбинации (depth 2)

- Дерево: `combine(primitive_a, primitive_b)` — min, max
- Глубина 2: два примитива + одна операция
- Генератор дерева → строка GLSL

**MVP:** Комнаты с двумя объектами (union/intersection).

### 4c: Глубина 1–2 по этажу

- Выбор глубины по floor (6–20 → 1–2)
- Полный набор примитивов: sphere, box, cylinder

**MVP:** На этаже 6 — простые, на 15 — чуть сложнее.

### 4d: Модификаторы (twist, repeat)

- Добавить twist, bend, repeat в грамматику
- Параметры из PRNG

**MVP:** Комнаты с искажёнными формами.

---

## Этап 5: Этажи и лестницы

**Цель:** Можно подниматься и спускаться.

**Задачи:**
- Этаж 0: вестибюль (одно пространство, узкий — пирамида) + лестница вверх
- Этаж 6: сетка комнат по формуле `rooms = floor(1 + 0.01 * 36) = 2`, side = 2 → сетка 2×2
- Лестницы: геометрия, триггер входа (подъём/спуск). Позиция по `hash(floor)` — в углу/на краю
- Переход между этажами: смена сцены, позиция у лестницы
- UI: номер этажа в углу

**MVP:** Старт на этаже 0 → поднимаюсь на 6 → хожу по комнатам → спускаюсь обратно.

---

## Этап 6: Этажи 1–5 и пустые ячейки

**Цель:** Полная иерархия этажей, «пустые» комнаты.

**Задачи:**
- **Этажи 1–5:** добавить генерацию (сейчас только 0 и 6). Одно большое пространство. **Упрощение:** один тип (длинный коридор) для всех; чередование (холл, парковка, колонны) — в этапе 9 или 10
- Этажи 6+: «пустые» ячейки — проём есть, за ним чёрный quad или пустой рендер; коллизия блокирует вход
- Детекция чёрного экрана: рендер 16×16, яркость < 0.02
- Retry: Room_ID+1… до 50 попыток при чёрном экране; при провале — ячейка «пустая»

**MVP:** Этажи 0–6 работают. Есть пустые проёмы и рабочие комнаты.

---

## Этап 7: Полная грамматика по зонам

**Цель:** Сложность растёт с этажом.

> Разбито на подэтапы 7a–7c.

### 7a: Зоны 21–50 (plane, torus, noise3D, rotate_y)

- Примитивы: + plane, torus
- Шум: noise3D (одна функция в шаблоне)
- Анимация: rotate_y(expr, uTime)
- Глубина 2–3

**MVP:** На этаже 25 — комнаты с шумом и вращением.

### 7b: Зоны 51–150 (fbm, fractal_repeat, pulse, slide)

- fbm, displace
- fractal_repeat
- pulse, slide
- Глубина 3–4

**MVP:** На этаже 80 — фрактальные и анимированные комнаты.

### 7c: Зоны 151+ (menger, sierpinski, morph) + цвет

- menger, sierpinski
- morph(expr_a, expr_b, t)
- Цвет в грамматике: градиенты HSV, пульсация hue

**MVP:** На этаже 200 — сложные комнаты с цветом.

---

## Этап 8: Кэш и стартовый экран

**Цель:** Не пересчитывать комнаты, сохранять прогресс.

**Задачи:**
- IndexedDB: store кэша `v{version}:{seed}:{floor}:{x}:{z}`
- При генерации — проверка кэша, при попадании — использовать
- **Гибрид:** при входе на этаж — предгенерация 10–20 ячеек от точки входа
- **LRU:** при переполнении (лимит 1 ГБ) — удаление по давности
- Стартовый экран: ввод seed, кнопка «Войти»
- Приоритет: URL > input > localStorage
- Сохранение позиции (этаж, x, z) в localStorage при смене
- Восстановление при перезагрузке

**MVP:** Перезагружаю страницу — остаюсь на том же этаже. Повторный заход в комнату — без ожидания.

---

## Этап 9: Освещение и детали

**Цель:** Атмосфера, лампы, индикатор.

**Задачи:**
- Коридоры: точечные лампы (позиции по hash), тёплый цвет
- Комнаты: ключевой свет с направления двери в шейдере
- Лампочка над лестницей — индикатор генерации (мерцание)
- Предзагрузка: при приближении к лестнице — генерация этажа в фоне

**MVP:** Коридоры освещены, при подъёме — лампочка мигает.

---

## Этап 10: Заархивировать и полировка

**Цель:** Сохранять понравившиеся комнаты, финальный UI.

**Задачи:**
- Кнопка «Заархивировать» в комнате (угол экрана)
- IndexedDB: store архива (код шейдера + метаданные)
- Меню по Escape: настройки (лимит кэша), выход
- WebGL fallback: сообщение при отсутствии поддержки
- Версионирование кэша: константа в коде

**MVP:** Могу сохранить понравившуюся комнату. Полный цикл: seed → исследование → архив.
