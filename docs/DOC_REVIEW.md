## 3. Неясности и риски

### 3.1 Реалистичность грамматической генерации

Генерация «всегда компилируемого» GLSL из дерева выражений — нетривиальная задача. Риск: много комнат с чёрным экраном или compile error.

**Добавлено:** SHADER_GRAMMAR — Parameter Guardrails (clamp радиусов, размеров для sphere, box, cylinder, torus, plane, repeat, twist/bend).

### 3.2 Порядок этапов: коридоры до вертикали

plan.md: сначала коридоры (2b), потом комнаты (3), потом этажи и лестницы (5).

README/ROADMAP: M1 = Vertical Loop (этажи 0–5), M2 = Procedural Corridors.

То есть в ROADMAP вертикаль идёт до процедурных коридоров 6+. В plan.md этажи 0–5 — в 5c, после коридоров и комнат.

**Рекомендация:** Явно зафиксировать порядок: что делаем первым — «ходьба + seed + этажи 0–5» или «ходьба + seed + коридоры 3×3». Для инкрементальной разработки логичен путь plan.md (сначала один этаж с коридорами, потом вертикаль).

### 3.3 Этаж 6: одна комната

Формула: floor(1 + 0.01*36) = 1 комната. Сетка 1×1. Один коридор, одна дверь.

plan.md: «Так задумано» — ок. Но для тестирования лабиринта нужна сетка минимум 2×2. Этап 2b: «сетка 3×3 (тест)».

**Вывод:** На этапе 2b используется фиксированная тестовая сетка 3×3, не формула. Формула включается в 5a. Согласовано.

## 4. Можно ли приступать к выполнению?

**Да, с оговорками.**

- Блокирующих открытых вопросов нет (questions.md).
- Архитектура, TECH_SPEC, DATA_MODEL, SHADER_GRAMMAR, UX_FLOWS, риски и тесты описаны.
- Перед стартом стоит:
  1. Разрешить противоречия (retry 20, cache 500 MB).
  2. Синхронизировать README и plan.md по этапу 0.
  3. Добавить `slide` в SHADER_GRAMMAR или убрать из README/plan.

Остальное можно уточнять по ходу (DFS-maze, пороги валидации, chunk/radius).

- 2a, 2b

- 5a, 5b, 5c, 5d
- 6
- 7a, 7b, 7c

**Что можно добавить:**

1. **Оценка объёма** (T-shirt: S/M/L) для каждого под-этапа — чтобы видеть относительный скоуп.
2. **Зависимости** — граф: 3b зависит от 3a, 4a зависит от 3b и т.д.
3. **Критерии «готово»** — уже есть MVP по этапам; можно добавить чеклист (например, для 4d: «retry работает», «empty кэшируется», «нет бесконечных циклов»).

**Рекомендация:** Добавить в plan.md колонку «Scope» (S/M/L) и опционально — граф зависимостей в начале документа.
