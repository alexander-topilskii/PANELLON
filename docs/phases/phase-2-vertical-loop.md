# Фаза 2 — Вертикальный loop [M1]

**Цель:** Создать фундамент структуры здания (многоэтажность), настроить специальные стартовые этажи (0-5) и реализовать механику переходов (лестницы).

## 2.1 Этажи 0 и 1–5 (Специальные шаблоны)
**Что делаем:**
- Определяем логическую модель Floor (`number`, `rooms`, `side` и флаг `isTemplate`).
- **Этаж 0:** Жестко заданный BoxGeometry/Mesh: вестибюль 6×6×2.5 м. Помещаем светящуюся площадку (меш) у дальней стены.
- **Этажи 1–5:** Жестко заданный прямой коридор 30×4×2.5 м. Меши пола, потолка и стен собираются в один объект. Лестницы размещаем на обоих концах коридора.
- Создаем HUD-элемент ("Этаж N" в углу экрана) для понимания текущей позиции.
- Вводим `FloorManager` или аналогичный контроллер для загрузки и выгрузки геометрии этажа из памяти.

**Детали реализации:**
- При смене этажа — `scene.remove(oldFloor)`, `oldFloor.dispose()`, чтобы не было утечек.
- Хранение текущего этажа в State-менеджере (связь с `TECH_SPEC.md`).

## 2.2 Лестницы и fade-переходы
**Что делаем:**
- **Геометрия лестницы:** PlaneGeometry (площадка на полу) с `MeshBasicMaterial` (светящийся цвет). В идеале наложить простейшую текстуру стрелки или просто цвет (зеленый - вверх, красный - вниз).
- **Логика триггера:** В Game Loop, используя `Box3` (AABB) игрока и площадки, проверяем пересечение.
- Если пересечение обнаружено -> блокируем управление (отключаем WASD), запускаем анимацию fade (например, через CSS `div` с `opacity`, перекрывающий canvas).
- После 0.3с "в темноте" -> увеличиваем/уменьшаем `currentFloor` -> генерируем новый этаж -> переносим игрока к стартовой лестнице нового этажа -> fade-in.

**MVP (Definition of Done):**
- Игрок стартует на 0 этаже.
- Идет до площадки, встает на нее -> экран темнеет -> игрок оказывается на 1 этаже.
- Может дойти до 5 этажа.
- Выполняет 20+ переходов туда-обратно без падения FPS и утечек памяти (проверить вкладку Memory в Chrome DevTools).
