# Фаза 3 — Процедурные коридоры [M2]

**Цель:** Создать ядро генерации лабиринта и геометрии коридоров для основных этажей (6+). Внедрить чанковую подгрузку (streaming) для больших сеткок.

## 3.1 Коридорные меши и коллизия (Ядро лабиринта)
**Что делаем:**
- **3.1a Data-слой:** Пишем алгоритм Randomized DFS (по `WORLD_GENERATION.md`). По входному Seed и Floor собираем `Uint8Array` с битовыми масками открытых стен для каждой ячейки 6x6. Начинаем тест на фиксированной сетке 3x3.
- **3.1b Render-слой:** Строим геометрию. Оптимизированный подход — использование `InstancedMesh` для стен, пола и потолка (чтобы не плодить тысячи Draw Calls).
  - Одна ячейка состоит из центральной "глухой" коробки 4x4 (будущая комната) и коридоров вокруг (ширина 2м).
  - Убираем/добавляем instance'ы стен между ячейками на основе битовой маски из Data-слоя.
- **3.1c Physics-слой:** Настраиваем Split-axis AABB Collision. Создаем массив Bounding Boxes для всех стен на этаже. Игрок движется по `dx`, проверяет пересечение `Box3`, при ударе `dx=0`; затем движется по `dz`, проверяет пересечение, при ударе `dz=0`.

## 3.2 Многоэтажность (6+)
**Что делаем:**
- Внедряем формулу роста этажей: `rooms = floor(1 + 0.01 * floor²)`.
- Вычисляем размер сетки: `side = ceil(sqrt(rooms))`.
- Центр сетки привязываем к (0,0,0) в Three.js координатах (World Space).
- Интегрируем размещение Лестницы (transition pad). Выбираем детерминированно одну ячейку на краю сетки для лестницы.

## 3.3 Chunked corridors (Стриминг мешей)
**Что делаем:**
- На этажах > 20x20 ячеек рендерить всё `InstancedMesh` тяжело (даже с instance'ами).
- Настраиваем логику чанков (Chunk Size: 16x16 ячеек). Топология лабиринта генерируется целиком (`Uint8Array` занимает килобайты, см. `WORLD_GENERATION.md`).
- В Game Loop: отслеживаем позицию игрока, определяем чанки в радиусе видимости (например, 2 чанка вокруг).
- При переходе границы чанка — строим `InstancedMesh` для нового чанка, удаляем (`dispose`) объекты старого чанка.

**MVP (Definition of Done):**
- Игрок на 10 этаже ходит по процедурно сгенерированному лабиринту. При перезапуске с тем же seed лабиринт идентичен.
- Игрок переходит на 100 этаж (сетка 11x11), коллизии работают, лаги отсутствуют.
- При движении по 500-му этажу (сетка ~50x50) меши подгружаются "на лету" (chunking) без заметных фризов.
