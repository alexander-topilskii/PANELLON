# PANELLON

## Что это за проект

**PANELLON** — веб-приложение (SPA), в котором вы исследуете бесконечный 3D-лабиринт в стиле лиминальных пространств и
«Гигахрущёвки». Всё генерируется процедурно, без участия человека.

Это **детерминированный цифровой монумент**: один и тот же сид всегда даёт один и тот же мир. Есть дефолтный
«официальный» мир.

https://alexander-topilskii.github.io/PANELLON/

### Визуальный стиль

- **Лиминальные пространства** — пустые коридоры, подъезды, переходы без людей
- **«Гигахрущёвка»** — бесконечный многоквартирный дом в духе советской архитектуры

---

## Концепция: «Вертикальный Архив Энтропии»

Бесконечная Гигахрущёвка, растущая вверх. Весь мир строится на основе **Seed-фразы** (зерна генерации).

**Детерминизм:** два человека с одним сидом видят идентичные комнаты, шейдеры и коридоры.

**Seed-фраза** — вводится при открытии мира (стартовый экран). **Приоритет:** URL (`?seed=...`) > поле ввода >
localStorage (последняя сессия). Без всего — дефолтный seed (например, `"official"`). При перезагрузке — восстановление
позиции (этаж, x, z) из localStorage.

---

## Навигация и управление

- **WASD** — движение, **мышь** — обзор камеры (pointer lock при клике на canvas)
- **Escape** — выход из pointer lock + меню (если открыто)
- **Физика** — персонаж ходит по полу (гравитация, коллизии split-axis: X → Z). Без прыжка и бега — минимализм
- **Лестницы** — светящаяся площадка (триггер-зона), вверх и вниз. Встал → fade → переход
- **Старт** — этаж 0, у входа (лестницы). При подъёме — выход у лестницы на следующем этаже

---

## Геометрия и рендеринг

- **Всё процедурно** — коридоры, комнаты, освещение, детали генерируются на лету
- **Коридоры** — процедурная геометрия (меши), см. ниже
- **Комнаты** — Raymarching: вся комната является искажённым пространством, шейдер рисует геометрию
- **Вход в комнату** — при пересечении проёма рендер переключается на fullscreen Raymarching-quad. Выход — обратно в
  коридор. Проём (дверная коробка) — в стене коридора. Превью-текстура (64×64, offscreen рендер) видна в проёме до
  входа. Ключевой свет в шейдере — со стороны «двери».

## Структура здания

- **Форма** — перевёрнутая пирамида: вершина внизу (этаж 0 узкий), основание вверху (этаж 9999 широкий)
- **Рост** — с подъёмом вверх количество комнат увеличивается. Формула только для этажей 6+:
  `rooms = floor(1 + 0.01 * floor²)` (k=0.01)
- **Планировка этажа** — сетка комнат, коридоры по DFS-maze (spanning tree, гарантированная связность). На краю —
  стена/тупик (не зациклено)
- **Сложность шейдера** — привязана к высоте (координата Y)
- **Масштаб** — человеческий (высота потолка ~2.5 м, комната ~4×4 м)
- **Верх здания** — 9999 этажей (на старте)

## Генерация шейдеров

- **Метод** — грамматическая генерация (деревья выражений), не перебор символов
- **Room_ID** → seed → детерминированная генерация дерева. Глубина по этажу (см. блок «Экспертиза»)
- **Успех** → можно зайти в комнату, результат кэшируется
- **Провал** → ячейка «пустая»: глухая стена, нет двери. Состояние хранится в `Map<"x,z", "valid" | "empty">` на этаж
- **Определение пустых ячеек** — гибрид (см. блок «Экспертиза»)

### Room ID

```
Room_ID = hash(GEN_VERSION, Global_Seed, Floor_Number, grid_x, grid_z)
```

`Room_Position` = (grid_x, grid_z) — координаты ячейки в сетке этажа. `GEN_VERSION` включён для изоляции несовместимых
грамматик (см. DETERMINISM_CONTRACT). Room_ID — seed для генератора. **Параметры примитивов** (радиусы, размеры) — из
Room_ID через seedable PRNG. **Hash:** mulberry32 — синхронный, детерминированный. **Seed-строка** → числовой Global_Seed
через hash строки (см. TECH_SPEC §1).

### Грамматика SDF

Дерево выражений, всегда дающее компилируемый GLSL:

- **Примитивы:** sphere, box, plane, torus, cylinder. Встраиваются в общий Raymarching-шаблон (uniform camera, raymarch
  loop, lighting).
- **Комбинации:** min, max, max(a,-b), smoothMin
- **Модификаторы:** twist, bend, repeat
- **Шум:** noise3D, fbm, displace(expr, noise)
- **Фракталы:** menger, sierpinski, fractal_repeat
- **Анимация:** rotate_y(expr, time), pulse, slide(expr, axis, time), morph(expr_a, expr_b, t)

### Цвет и освещение в шейдерах

Отдельная логика в грамматике: цвет зависит от позиции, нормали, расстояния. Варианты — градиенты (HSV по углу),
пульсирующие оттенки (`sin(time) * hue`), шум для «грязного» цвета. Всё процедурно, без текстур.

## Система фильтрации и сохранения

### Авто-валидация

Грамматика даёт компилируемый код. Если результат — чёрный экран (вырожденная SDF), система пробует альтернативное
дерево (Room_ID + 1, +2… до успеха, лимит 20 попыток; при провале — ячейка «пустая»). **Детекция:** рендер в малый
буфер (16×16), средняя яркость < 0.02 и variance < 0.001 — считаем чёрным. При ошибке компиляции — fallback на простой gradient.

### Заархивировать

Кнопка появляется при нахождении в комнате (угол экрана или по наведению). Сохраняет код шейдера + метаданные (комната,
этаж) в IndexedDB. Планируется онлайн-галерея «лучших находок».

## Технологии

- **Web** — SPA, доступ по ссылке, без установки. Десктоп в приоритете; мобильные — touch для обзора, при необходимости.
  При отсутствии WebGL — сообщение об ошибке.
- **Three.js** — WebGL, рендеринг мешей (коридоры) и кастомных шейдеров (комнаты)
- **IndexedDB** — два store: кэш (результаты генерации) и архив «золотых» (по кнопке). Лимит 500 МБ на кэш. LRU — только
  кэш; архив не трогаем. **Версионирование:** ключ кэша `v{version}:{seed}:{floor}:{x}:{z}`; версия — константа в коде,
  при смене грамматики инкремент.
- **UI** — минималистичный. Seed — стартовый экран. Номер этажа — угол экрана («Этаж 42»). Настройки (кэш и др.) —
  иконка шестерёнки или меню по Escape

## Производительность и UX

- Рисуем **одну комнату** за раз (fullscreen quad). Целевые 60 FPS. **Adaptive resolution:** Raymarching рендерится в
  буфер (0.5–1.0× экрана), upscale — сложные шейдеры на верхних этажах работают за счёт снижения разрешения.
- **Коридоры:** на больших этажах (сетка > 20×20) — chunked rendering: меши только в радиусе ~20 ячеек от игрока.
- При входе на этаж — генерация + fade ~0.3 с. Предзагрузка при приближении к лестнице.
- При переходе — dispose мешей, материалов и текстур предыдущего этажа (одновременно в памяти только один этаж).

## Детали реализации (экспертиза)

### Пространственная модель

Игрок ходит по **коридорам**. Комнаты — за дверями в стенах коридоров.

```
    6 м          6 м
 ┌──────────┬──────────┐
 │ ┌──────┐ │ ┌──────┐ │
 │ │      │ │ │      │ │  6 м
 │ │ room │ │ │ room │ │
 │ │ 4×4  │д│ │ 4×4  │ │
 │ └──────┘ │ └──────┘ │
 │  corridor  corridor │
 ├──────────┼──────────┤
 │ ┌──────┐ │ ┌──────┐ │
 │ │      │ │ │      │ │  6 м
 │ │ room │ │ │ room │ │
 │ │ 4×4  │ │ │ 4×4  │д│
 │ └──────┘ │ └──────┘ │
 │  corridor  corridor │
 └──────────┴──────────┘
  д = дверь (проём в стене коридора)
```

- **Ячейка:** 6×6 м. Комната 4×4 м в центре, коридор 2 м по периметру
- **DFS-maze** определяет, какие стены между ячейками убраны (= проходы между коридорами)
- **Проход ≠ дверь.** Проход — убранная стена (maze). Дверь — проём в стене коридора, ведущий в raymarching-комнату.
  Дверь ставится со стороны одного из открытых проходов ячейки (выбор по hash)
- **Seed:** `hash(Floor + Segment_Position + Global_Seed)`. Вариации по hash: высота, наличие труб/проводов

### Специальные этажи

- **Этаж 0:** вестибюль 6×6×2.5 м. Пустая комната, лестница вверх у дальней стены
- **Этажи 1–5:** прямой коридор 30×4×2.5 м, голые стены, тусклый свет. Лестницы на обоих концах. Без сетки, без комнат

### Освещение

- **Коридоры:** точечные лампы, позиции по `hash(floor, segment_id)` — детерминированно. Цвет: тёплый (жёлтый) для
  лиминальной атмосферы, лёгкая вариация по этажу. На этажах 1–5 — возможное мерцание.
- **Комнаты:** освещение внутри Raymarching-шейдера — ambient + ключевой свет с направления «двери» (чтобы не было
  плоского).

### Гибрид: предпроверка + на лету

При входе на этаж — предгенерация комнат в радиусе 15 ячеек от точки входа. При приближении к лестнице — prewarm зоны
следующего этажа. При приближении к ячейке вне зоны — генерация на лету. Кэшируем и «пустые» (нет двери), и «успешные»
(шейдер) — повторно не генерируем.

### Рост числа комнат и сетка

Только этажи 6+. Формула: `rooms_on_floor = floor(1 + 0.01 * floor²)`. Примеры: этаж 10 → 2, этаж 100 → 101, этаж 500 →
2501. **Сетка:** квадратная, сторона = ceil(sqrt(rooms)). grid_x, grid_z от 0 до side-1. Центр этажа — (0, 0) в мировых
координатах.

### Глубина дерева (грамматика)

| Этажи   | Глубина |
|---------|---------|
| 6–20    | 1–2     |
| 21–50   | 2–3     |
| 51–150  | 3–4     |
| 151–500 | 4–5     |
| 500+    | 5–8     |

### Лестницы

Одна лестница на этаж. Позиция — по краю сетки или в углу, детерминированно от `hash(floor)`. Пройти мимо 3–5 комнат
до лестницы. Лестницы ведут вверх и вниз.

**Геометрия:** светящаяся площадка на полу (emissive plane) + стрелка вверх/вниз. Без ступенек.
**Взаимодействие:** триггер-зона — игрок встаёт на площадку → fade-out 0.3 с → dispose этажа → генерация нового →
fade-in. Спавн у лестницы нового этажа.

## Референсы (стиль)

Визуальные ориентиры для процедурной генерации. **Всё в игре генерируется процедурно** — референсы задают атмосферу, а
не копируются.

| Элемент                    | Референсы                                                                      |
|----------------------------|--------------------------------------------------------------------------------|
| **Коридоры**               | Backrooms, Control, фото советских подъездов — стены, освещение, пустота       |
| **Технический этаж (1–5)** | Провода, трубы, тусклые лампы, «бытовой мусор» лиминальных пространств         |
| **Гигахрущёвка**           | Atomic Heart, панельная архитектура — масштаб, повторяемость, типовые элементы |

## Принятые решения (ответы на ключевые вопросы)

### 1) Бесконечность vs 9999 этажей

На первом этапе проект считается **практически бесконечным**, но технически ограниченным:

- **Текущее ограничение:** `MAX_FLOOR = 9999` (для MVP/v1)
- **Концепция:** мир «бесконечный» в художественном смысле
- **Технический путь к настоящей бесконечности:** переход к windowed-генерации этажей и хранению только активного диапазона

### 2) Лестницы (устранение конфликта)

Единое правило:

- **Этаж 0:** только лестница вверх
- **Этажи 1-5:** две лестницы (вверх и вниз) на противоположных концах прямого коридора
- **Этажи 6+:** две площадки (вверх/вниз) в одной точке, позиция по hash(floor)

### 3) Реалистичный масштаб генерации

Формула роста сохраняется, генерация разделена на два уровня (ADR-0007):

- **Maze topology — eager:** DFS-maze генерируется целиком при входе на этаж (компактные данные, < 5 мс для этажей до 1000)
- **Corridor meshes — chunked:** меши строятся только для активного радиуса (~20 ячеек от игрока)
- На этажах ≤ 20×20 — рендерим всё сразу

### 4) Детерминизм: точная формулировка

Гарантируется:

- одинаковая топология, Room_ID, выбор грамматики и параметры генерации для одинакового seed
- одинаковое поведение сохранений/кэша в рамках одной версии генератора

Не гарантируется побитовая идентичность картинки на разных GPU/драйверах, но визуально сцены должны быть эквивалентны.

### 5) Пустые комнаты и ретраи

Правило:

- до `N=20` ретраев генерации
- затем fallback: «empty» (глухая стена без входа)
- «чёрный экран» определяется комбинацией метрик (brightness + variance), чтобы не отбрасывать валидные тёмные сцены

## Vision (кратко)

SPA с бесконечным процедурным 3D-лабиринтом. Всё генерируется процедурно: коридоры, комнаты, детали. От первого лица,
физика. Seed при открытии мира. Коридоры — меши (Three.js), комнаты — Raymarching с GLSL-шейдерами из грамматической
генерации. Шум, фракталы, анимация, цвет. Референсы: Backrooms, Control, Atomic Heart, советские подъезды. Сложность
растёт с высотой. Детерминированность по Seed-фразе. Звука пока нет.
